Introducción

Bienvenido al manual de usuario y técnico de SIGB, el Sistema Integral de Gestión de Bibliotecas que automatiza y centraliza la administración completa de una biblioteca.

¿Qué es SIGB?

SIGB es una aplicación de escritorio diseñada en Java que cubre el ciclo completo de gestión bibliotecaria:

  • Gestión centralizada de catálogo bibliográfico
  • Sistema de préstamos y devoluciones automatizado
  • Control de inventarios físicos
  • Módulo de reservas
  • Gestión de multas y caja
  • Seguridad con autenticación biométrica
  • Portal del cliente (autoservicio)
  • Reportes financieros y estadísticos

Característica clave: El sistema está segmentado por roles para garantizar que cada usuario acceda únicamente a las funciones que le corresponden, mejorando la seguridad y eficiencia operativa.

Instalación y Configuración (Usuario)

1 Instalación del JRE

Asegúrese de tener instalado Java Runtime Environment (JRE) 8 o superior. Si no lo tiene, descárguelo e instálelo desde el sitio oficial de Oracle.

2 Ejecución del Sistema

El sistema se ejecuta a través del archivo ejecutable principal (generalmente un archivo .jar o un wrapper de escritorio).

3 Configuración Inicial (Solo en Primer Uso)

Si el archivo de configuración de la conexión a la base de datos no existe, el sistema solicitará la ruta de conexión a la base de datos SQL Server. Esta información es proporcionada por el Administrador de Sistemas.

Operación del Sistema

Inicio de Sesión y Autenticación

El sistema requiere autenticación de dos factores o primaria avanzada.

Acceso Estándar

1 Login

Ingrese su Nombre de Usuario y Contraseña en los campos correspondientes.

2 Iniciar Sesión

Haga clic en "Iniciar Sesión" para acceder al sistema.

Opción Avanzada (Reconocimiento Facial)

1 Inicio de Sesión Facial

Si su rostro está enrolado en el sistema, puede utilizar el botón "Iniciar con Reconocimiento Facial" para un acceso más rápido.

2 Requisitos

Asegúrese de tener una cámara conectada y una buena iluminación.

Proceso de Enrolamiento Facial (Registro Biométrico)

1 Registro Inicial

Si es la primera vez que inicia sesión con esta función, el Administrador o Bibliotecario deberá realizar su registro facial.

2 Enrolamiento

El sistema le solicitará de 3 a 5 capturas faciales para crear su plantilla biométrica.

3 Recomendaciones

Mantenga una expresión neutral y una iluminación estable mientras se toman las capturas para asegurar la precisión.

Menú Principal y Dashboard

Una vez autenticado, el sistema le mostrará el menú principal con las opciones permitidas para su rol.

El menú lateral izquierdo le permite acceder a los módulos de Catálogo, Inventarios, Préstamos/Devoluciones, Multas, Reportes y Seguridad. Las opciones visibles dependerán de su rol asignado.

Módulo Catálogo (Administrador y Bibliotecario)

Permite la gestión de los recursos de la biblioteca.

Autores/Categorías/Editoriales (CRUD)

El Administrador puede realizar las operaciones CRUD (Crear, Leer, Actualizar, Eliminar) en el maestro de autores y categorías.

Libros (CRUD)

Permite registrar libros con metadatos clave: ISBN, edición, año, idioma, editorial, y la ubicación física (sala, estante, nivel).

Búsqueda Avanzada

Utilice la función de búsqueda con filtros combinados por título, autor, categoría, rango de años y disponibilidad para localizar material rápidamente.

Módulo Inventarios (Administrador y Bibliotecario)

Gestiona las copias físicas de los libros.

Gestión de Libros

Permite dar de alta/baja a las copias individuales (usando baja lógica) y reasignar su ubicación.

Gestión de Movmientos de inmventario

Permite dar a conocer movimientos en en inventario de libros.

Vitacora de Inventario

El sistema soporta conteos cíclicos y el ajuste de existencias con una bitácora de diferencias.

Alertas de inventario

Se generan avisos por mínimos de copias disponibles o por libros extraviados/deteriorados.

Módulo Préstamos y Devoluciones (Bibliotecario)

Gestiona el flujo de material de la biblioteca.

Proceso de Préstamo

1 Selección de Cliente

Use el campo de búsqueda para localizar al cliente. El sistema validará automáticamente que el cliente no esté bloqueado por morosidad o haya excedido el límite de préstamos activos.

2 Selección de Copia

El cliente puede reservar un libre sin afectar el inventario en el cual coloca el numero de cola que tiene el cliente para obtener el libro, luego de realizar la reserva se procede a ingresar el presatamo.

3 Confirmación

El sistema muestra las reservas del cliente selccionado se valida los datos y se registra y muestra la fecha en que se puede entregar los libro(15 dias).

Proceso de Devolución

1 Registro de Devolución

selecciona el cliente y desplegara el listado de los prestamos que desa devolver, se le habilitara un campo de fecha dodne deberia de entregarlo y la fecha donde lo entrego y le habilita un checkbos aplica a la multa.

2 Cálculo de Multa

evalua la fecha de prestamo vs de devolucion en la cual automaticamente hace el ajuste del valor de la multa.

Renovaciones

Permite extender la fecha de vencimiento si no hay reservas pendientes para ese título.

Módulo Multas y Caja (Módulo Financiero)

Gestiona las cuentas por cobrar y el ingreso de multas.

Cobro de Multas

1 Consulta de Multas

La vista de multas muestra todas las multas pendientes de pago generadas al momento de la devolución tardía, indicando el monto total a cobrar.

2 Cobro en Caja

Seleccione la multa a pagar. El módulo de Caja permite registrar el cobro y genera un recibo.

Gestión de Caja

1 Caja Diaria Al finalizar la jornada, el usuario financiero puede realizar el Arqueo y Cierre de Caja, generando un resumen de los ingresos del día.

2 Políticas de Multa

Las reglas de multa (monto por día de atraso, tope) son parametrizables por el Administrador. Se pueden aplicar exoneraciones con justificación.

Módulo Cliente (Autoservicio)

Mini-aplicación para uso del lector tras su autenticación.

Resumen Personal

Una vez que el cliente inicia sesión, puede ver un resumen de sus:

  • Préstamos Vigentes (con fecha de vencimiento)
  • Reservas Activas
  • Historial de Multas pendientes o pagadas
  • Facturas pagas y por pagar
  • Devoluciones activas

Consulta de Catálogo

El cliente puede buscar libros por título, autor o categoría y revisar la disponibilidad en tiempo real.

Objetivo y Alcance (Manual Técnico)

Objetivo

Este manual técnico documenta los aspectos técnicos del Sistema Integral de Gestión de Biblioteca (SIGB), incluyendo su arquitectura, diseño de base de datos, y estándares de codificación. Esto facilita a los analistas, programadores y administradores de sistemas el mantenimiento, la modificación y las futuras actualizaciones del sistema.

Alcance

La aplicación es un sistema de escritorio desarrollado en Java, con persistencia en SQL Server, implementando una gestión integral de biblioteca que abarca desde el catálogo y préstamos hasta la seguridad biométrica y la reportería.

Arquitectura del Sistema

Modelo de Arquitectura

El SIGB implementa la arquitectura de Modelo Vista Controlador (MVC), lo que garantiza la separación de responsabilidades y facilita la mantenibilidad del código:

Componente Responsabilidad
Modelo (M) Contiene la lógica de negocio, las entidades de datos (POJOs) y las capas de acceso a datos (DAO - Data Access Objects). Es responsable de interactuar con SQL Server.
Vista (V) Implementada con Java Swing y diseñada a través de un entorno visual (como IntelliJ UI Designer). Contiene la interfaz gráfica que ve el usuario.
Controlador (C) Maneja las interacciones del usuario (eventos de botones, mouse) y coordina la actualización del Modelo y de la Vista.

Seguridad y Estándares de Codificación

Seguridad de Contraseñas y Biometría

Hash BCrypt: Las contraseñas de los usuarios se almacenan utilizando la función de Hash BCrypt para asegurar que nunca se guarden en texto plano.

Reconocimiento Facial: La autenticación biométrica se realiza mediante:

  • Enrolamiento: Captura de 3 a 5 imágenes para generar una plantilla que se guarda en la tabla RostroUsuario. No se almacenan las fotos crudas del usuario.
  • Login Facial: La plantilla capturada en el momento del login se compara con la guardada.

Auditoría: Se implementa un registro detallado en la tabla Auditoria para todas las acciones sensibles (ej. alta de copias, modificación de multas, login exitoso/fallido).

Estándares de Codificación

  • Patrón MVC: El código fuente está estructurado para cumplir con el patrón MVC, separando las clases de Vista, Controlador y DAO (Data Access Objects).
  • Prevención de Inyección SQL: Todas las interacciones con la base de datos deben utilizar PreparedStatement de Java para parametrizar las consultas y prevenir vulnerabilidades de inyección SQL.
  • Gestión de Conexión: La capa de conexión a la base de datos (ODBCConnection o similar) es centralizada para facilitar la gestión de recursos.

Instalación y Despliegue

1 Script de Base de Datos

Ejecutar el script db_schema.sql (o el equivalente) en el servidor de SQL Server para crear la base de datos, todas las tablas e índices.

2 Configuración de Conexión

La cadena de conexión a la base de datos debe ser almacenada en un archivo de configuración seguro (ej. un archivo .properties o .xml), fuera del código fuente, para no exponer credenciales.

3 Compilación y Ejecución

Compilar el proyecto con Maven para generar el archivo ejecutable (.jar) y distribuirlo a las estaciones de trabajo cliente.

4 Permisos de Usuario DB

El usuario de conexión a la base de datos debe tener, como mínimo, los permisos de SELECT, INSERT, UPDATE, y DELETE sobre las tablas del esquema.

// Ejemplo de configuración de conexión en application.properties
database.url=jdbc:sqlserver://localhost:1433;databaseName=SIGB
database.username=sigb_user
database.password=contraseña_segura

Mantenimiento y Evolución

Mantenimiento de Módulos

La estructura MVC permite que el mantenimiento se centre en las clases del Modelo y los DAO correspondientes.

  • Para cambios en las reglas de negocio (ej. cálculo de multas), modificar la clase de servicio que interactúa con la tabla Parametro.
  • Para añadir una nueva funcionalidad, se debe crear una nueva Vista, un nuevo Controlador y extender los DAO existentes.

Escalabilidad

El sistema está diseñado para permitir futuras adiciones como un soporte para múltiples idiomas o una migración a una arquitectura de microservicios sin reescribir la lógica de negocio (Modelo).

Reportes

El módulo de reportes es independiente y permite agregar nuevos informes (PDF/Excel) utilizando filtros por fecha, usuario y ubicación.

Solución de Problemas Comunes

Mensaje de Error Posible Causa Solución
"Error en la cadena de conexión o SQL Server no disponible." El servidor de base de datos está inactivo o la configuración de conexión es incorrecta. Verifique la conectividad de red y contacte al Administrador de Sistemas.
"Credenciales no válidas." Usuario o contraseña incorrectos. Intente de nuevo o use la función de restablecimiento de contraseña (si está disponible).
"El usuario ha excedido el límite de préstamos." El cliente tiene el máximo de libros permitidos o tiene multas sin pagar que lo bloquean. Solicite la devolución de material o el pago de las multas pendientes.
"Fallo en la verificación facial: Iluminación o rostro no encontrado." Mala iluminación, rostro cubierto o plantilla biométrica no enrolada. Asegúrese de tener buena luz y repita el proceso de enrolamiento si es necesario.

Nota: Para problemas técnicos complejos, contacte al administrador del sistema o al equipo de desarrollo.

Seguridad en la Aplicación Móvil

1 Autenticación con JWT

Las credenciales se validan una vez y se emite un token JWT con expiración que se usa en requests posteriores.

2 Autorización por Usuario

Cada usuario solo puede acceder a sus propios datos. El backend valida que el ID de usuario en el token coincida con los datos solicitados.

3 Comunicación Segura

Todas las comunicaciones utilizan HTTPS con certificados SSL válidos.

4 Almacenamiento Local Seguro

El token JWT se almacena de forma segura usando Keychain (iOS) o Keystore (Android).

Flujo de Sincronización

  1. La aplicación móvil realiza login y obtiene token JWT
  2. Se descargan los datos del usuario (préstamos, reservas, multas)
  3. Los datos se almacenan localmente para acceso offline
  4. La aplicación verifica periódicamente actualizaciones
  5. Las notificaciones push alertan sobre cambios importantes
